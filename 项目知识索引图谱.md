## 三、核心数据流转

```
                    人工编辑
                       ↓
              ┌─────────────────┐

              └────────┬────────┘
                       │  npm run sync
                       ↓
              ┌─────────────────┐
              │ src/initialData.ts│  ← 自动生成，勿手动改
              └────────┬────────┘
                       │  npm run build
                       ↓
              ┌─────────────────┐
              │  dist/ (静态页面) │
              └────────┬────────┘
                       │  git push origin master
                       ↓
              ┌─────────────────┐
              │   GitHub Pages   │  → 线上自动更新
              └─────────────────┘
```

### 日常更新简历的3步操作

```bash
# 1. 添加所有修改
git add -A

# 2. 提交
git commit -m "更新简历"

# 3. 推送（约1分钟后自动部署生效）
git -c http.sslBackend=openssl push origin master
```

> **注意**：推送前不需要手动运行 `npm run sync` 或 `npm run build`，GitHub Actions 会自动执行。

## 四、Excel 数据结构详解

`resume-data.xlsx` 包含 7 个 Sheet，全部使用**中文列名**：

| Sheet 名 | 用途 | 关键列 |
|-----------|------|--------|
| 📋使用说明 | 操作指南 | 模块、说明、填写提示 |
| 👤个人信息 | 姓名/职位/联系方式等 | 字段名（勿改）、内容（请修改这里） |
| 🎯技能列表 | 每行一条技能 | 技能名称、熟练度(0-100)、分类、经验年限 |
| 🏢工作经历 | 每行一段工作经历 | 编号、公司名称、职位、开始/结束时间、工作描述、技能标签 |
| 📊项目经历 | 每行一个项目 | 编号、项目名称、项目类别、是否重点、项目角色、项目周期、项目简介、背景、目标、关键行动、量化成果、亮点摘要、技术/方法论 |
| 🎓教育背景 | 每行一条教育/资格 | 编号、类别（学历教育/职业资格/培训认证）、学校/机构、学历、专业、证书名称、亮点 |
| ⚙GitHub配置 | 概览页展示数据 | 配置项（勿改）、值（请修改这里） |

### Excel 填写规则

- **多条内容用英文分号 `;` 分隔**（不要用中文分号）
- **日期格式**：`YYYY-MM`（如 `2024-09`），至今填 `至今`
- **技能分类**只填这4个：`薪酬福利规划` / `HRBP业务协同` / `组织效能与绩效激活` / `人力数字化与用工合规`
- **教育类别**只填这3个：`学历教育` / `职业资格` / `培训认证`
- **是否重点**填 `是` 或 `否`
- **以 `📝` 开头的行**是提示行，同步时自动跳过
- **个人信息和GitHub配置**：只改「内容/值」列，别动「字段名/配置项」列

## 五、类型系统（types.ts）

```
ResumeData
 ├── version: string              # 数据版本号
 ├── _version?: string            # 缓存版本号（用于 localStorage 失效）
 ├── personalInfo: PersonalInfo   # 个人信息（12个字段）
 ├── skills: Skill[]              # 技能列表
 ├── experience: Experience[]     # 工作经历
 ├── projects: Project[]          # 项目经历（含展开详情）
 ├── education: Education[]       # 教育背景
 └── githubConfig: GitHubConfig    # GitHub 展示配置
```

### 核心接口字段速查

| 接口 | 关键字段 | 说明 |
|------|----------|------|
| **PersonalInfo** | name, title, avatarUrl, email, phone, location, bio, summary | bio 是长介绍，summary 是3句精炼概述 |
| **Skill** | name, level(0-100), category(4选1), yearsOfExp | category 为中文枚举 |
| **Experience** | id, company, role, startDate, endDate, description[], techStack[] | description 和 techStack 是字符串数组 |
| **Project** | id, title, description, featured, highlights[], background?, objective?, actions[], outcomes[], role?, timeline? | 带?的是展开详情字段，卡片点击后展示 |
| **Education** | id, institution, degree, major, category(3选1), certName?, certOrg?, highlights[] | 职业/培训类需填证书信息 |
| **GitHubConfig** | username, useLiveApi, repoCountOverride, languages[] | 概览页展示的数据，可覆盖真实 API 返回值 |

 │   │   ├── 专业技能：分类筛选 + 技能卡片网格
 │   │   ├── 工作经历：时间线布局 + 展开详情
 │   │   ├── 核心项目：项目卡片 + 展开详情（背景/目标/行动/成果）
 │   │   ├── 教育背景：教育/资格卡片

### 关键机制

| 机制 | 说明 |
|------|------|
| **暗色模式** | 通过 `darkMode` state 切换，Tailwind `dark:` 前缀实现 |
| **localStorage 缓存** | 页面数据存入 localStorage，刷新不丢失 |
| **版本号失效** | `_version` 字段比对，Excel 更新推送后自动丢弃旧缓存 |
| **打印适配** | `print:hidden` 隐藏导航/按钮，`print:block` 显示 A4 排版简历 |
| **移动端适配** | 左侧栏在手机端隐藏，通过汉堡菜单切换显示 |

## 七、脚本说明

### `npm run sync`（sync-data.mjs）

**作用**：读取 `resume-data.xlsx` → 生成 `src/initialData.ts`

**流程**：
1. 读取 Excel 的 7 个 Sheet
2. 自动跳过以 `📝` 开头的提示行
3. 将中文列名映射为英文字段名（通过 COL_MAP）
4. 用英文分号 `;` 拆分多条内容字段
5. 技能分类映射（支持旧英文值兼容转中文）
6. 自动生成 `_version` 日期版本号
7. 输出 `initialData.ts`

### `npm run build`

等价于 `npm run sync && vite build`，先同步数据再构建。

### `node scripts/create-template.mjs`

重新生成空白 `resume-data.xlsx` 模板（会覆盖现有数据，慎用）。

## 十、注意事项

1. **`src/initialData.ts` 是自动生成文件**，每次 `npm run sync` 会覆盖，不要手动修改
2. **推送用 `git -c http.sslBackend=openssl push`**，直接 `git push` 可能因 VPN/代理报 SSL 443 错误
3. **GitHub Pages 部署约需1分钟**，推送后稍等再刷新线上页面
4. **头像路径**必须是 `/interactive-visual-resume/avatar.jpg`（含子路径前缀），否则 GitHub Pages 无法加载
5. **Excel 中不要用中文引号 `""`**，会导致 JS 解析报错，如需引号用直角引号 `「」`
6. **Excel 中数字字段不要填 `-`**，必须填实际数字，否则同步脚本报错
7. **推送前建议先 `npm run build` 验证**，确保构建无报错
8. **`.bak` 文件可以删除**，是修改前的备份
